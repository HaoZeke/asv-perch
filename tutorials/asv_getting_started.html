Getting Started¶

This guide walks through setting up asv-perch in a project that already has ASV benchmarks and a CI workflow that runs them on pull requests.

Prerequisites¶

  • A GitHub repository with ASV benchmarks

  • A CI workflow (“Benchmark PR”) that:

    • Runs benchmarks for both the base (main) and PR head commits

    • Uploads results as a workflow artifact

  • uv available on the runner (via astral-sh/setup-uv)

Workflow Structure¶

The typical setup uses two workflows:

  1. Benchmark PR – triggered on pull_request, runs the benchmarks, uploads results as an artifact

  2. Comment Benchmark Results – triggered by workflow_run on the benchmark workflow, downloads the artifact and posts the comment

This two-workflow pattern is required because pull_request workflows from forks do not have write access to the repository. The workflow_run event runs in the context of the base repository with full permissions.

Benchmark Workflow¶

Your benchmark workflow produces an artifact containing ASV result JSON files, metadata, and optionally a pre-computed comparison. There are two patterns:

Pattern 1: Matrix Strategy (Recommended)¶

Use a matrix to run base and PR benchmarks in parallel on separate runners. A lightweight combine job runs asv-spyglass compare and bundles everything.

name: Benchmark PR
on:
  pull_request:
    branches: [main]

jobs:
  benchmark:
    strategy:
      fail-fast: false
      matrix:
        include:
          - label: main
            sha: ${{ github.event.pull_request.base.sha }}
          - label: pr
            sha: ${{ github.event.pull_request.head.sha }}
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - name: Checkout target
        run: |
          mkdir -p /tmp/_bench_preserve
          cp -r benchmarks/ /tmp/_bench_preserve/
          git checkout -f ${{ matrix.sha }}
          git clean -fd
          cp -r /tmp/_bench_preserve/benchmarks/ benchmarks/
      - name: Run benchmarks
        run: |
          asv machine --yes
          asv run --record-samples --set-commit-hash ${{ matrix.sha }}
      - uses: actions/upload-artifact@v4
        with:
          name: bench-${{ matrix.label }}
          path: .asv/results/

  combine:
    needs: benchmark
    runs-on: ubuntu-latest
    steps:
      - uses: astral-sh/setup-uv@v5
      - uses: actions/download-artifact@v4
        with:
          name: bench-main
          path: results/
      - uses: actions/download-artifact@v4
        with:
          name: bench-pr
          path: results/
      - name: Compare
        run: |
          BASE_PREFIX="${{ github.event.pull_request.base.sha }}"
          PR_PREFIX="${{ github.event.pull_request.head.sha }}"
          BASE_FILE=$(find results -name "${BASE_PREFIX:0:8}*.json" | head -1)
          PR_FILE=$(find results -name "${PR_PREFIX:0:8}*.json" | head -1)
          uvx --from "git+https://github.com/HaoZeke/asv_spyglass.git@enh-multiple-comparisons" \
            asv-spyglass compare "$BASE_FILE" "$PR_FILE" \
            --label-before main --label-after pr > results/comparison.txt
          echo "main_sha=${{ github.event.pull_request.base.sha }}" > results/metadata.txt
          echo "pr_sha=${{ github.event.pull_request.head.sha }}" >> results/metadata.txt
      - uses: actions/upload-artifact@v4
        with:
          name: benchmark-results
          path: results/

Pattern 2: Single Job¶

Run both benchmarks sequentially in one job. Simpler but slower.

The artifact should contain:

benchmark-results/
  machine_name/
    a1b2c3d4-envhash.json   # base commit results
    e5f6g7h8-envhash.json   # PR commit results
  benchmarks.json            # benchmark metadata
  metadata.txt               # SHA mapping
  comparison.txt             # optional pre-computed comparison

The metadata.txt file should contain:

main_sha=a1b2c3d4e5f67890abcdef1234567890abcdef12
pr_sha=e5f6g7h890abcdef1234567890abcdef12345678

Both main_sha= and base_sha= keys are supported for the base commit.

Commenter Workflow¶

Create .github/workflows/ci_bench_commenter.yml:

name: Comment benchmark results

on:
  workflow_run:
    workflows: ["Benchmark PR"]
    types: [completed]

jobs:
  comment:
    if: >-
      github.event.workflow_run.event == 'pull_request' &&
      github.event.workflow_run.conclusion == 'success'
    runs-on: ubuntu-latest
    permissions:
      pull-requests: write
      issues: write
      actions: read

    steps:
      - name: Set up uv
        uses: astral-sh/setup-uv@v5

      - name: Download benchmark artifact
        uses: actions/download-artifact@v4
        with:
          name: benchmark-results
          path: results
          run-id: ${{ github.event.workflow_run.id }}
          github-token: ${{ secrets.GITHUB_TOKEN }}

      - name: Post benchmark comment
        uses: HaoZeke/asv-perch@v1
        with:
          github-token: ${{ secrets.GITHUB_TOKEN }}
          results-path: results/
          metadata-file: results/metadata.txt
          regression-threshold: '10'
          runner-info: ubuntu-latest

Required Permissions¶

The commenter workflow needs:

Permission

Reason

pull-requests: write

Post/update PR comments

issues: write

Required for comment API

actions: read

Download artifacts from other runs

What Happens Next¶

When a pull request triggers the benchmark workflow and it succeeds:

  1. The commenter workflow starts automatically

  2. It downloads the benchmark results artifact

  3. Reads SHAs from metadata.txt

  4. Finds the matching JSON files by SHA prefix

  5. Runs asv-spyglass compare (or reads comparison.txt)

  6. Parses the output into regressions, improvements, unchanged

  7. Renders a formatted comment with emoji indicators

  8. Posts (or updates) the comment on the PR

  9. Writes the same content to the workflow step summary

If the same PR is updated and benchmarks run again, the existing comment is updated in place (idempotent via an HTML marker).
