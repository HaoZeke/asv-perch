
===============
Getting Started
===============

This guide walks through setting up ``asv-benchmark-commenter`` in a project that
already has ASV benchmarks and a CI workflow that runs them on pull requests.

Prerequisites
-------------

- A GitHub repository with ASV benchmarks

- A CI workflow ("Benchmark PR") that:

  - Runs benchmarks for both the base (main) and PR head commits

  - Uploads results as a workflow artifact

- ``uv`` available on the runner (via ``astral-sh/setup-uv``)

Workflow Structure
------------------

The typical setup uses two workflows:

1. **Benchmark PR** -- triggered on ``pull_request``, runs the benchmarks, uploads
   results as an artifact

2. **Comment Benchmark Results** -- triggered by ``workflow_run`` on the benchmark
   workflow, downloads the artifact and posts the comment

This two-workflow pattern is required because ``pull_request`` workflows from forks
do not have write access to the repository. The ``workflow_run`` event runs in the
context of the base repository with full permissions.

Benchmark Workflow
------------------

Your benchmark workflow should produce an artifact containing:

- ASV result JSON files (one per commit, named by SHA prefix)

- A ``metadata.txt`` file with the base and PR SHAs

- Optionally, a pre-computed ``comparison.txt``

Example artifact structure:

::

    benchmark-results/
      machine_name/
        a1b2c3d4-envhash.json   # base commit results
        e5f6g7h8-envhash.json   # PR commit results
      benchmarks.json            # benchmark metadata
      metadata.txt               # SHA mapping
      comparison.txt             # optional pre-computed comparison

The ``metadata.txt`` file should contain:

::

    main_sha=a1b2c3d4e5f67890abcdef1234567890abcdef12
    pr_sha=e5f6g7h890abcdef1234567890abcdef12345678

Both ``main_sha=`` and ``base_sha=`` keys are supported for the base commit.

Commenter Workflow
------------------

Create ``.github/workflows/ci_bench_commenter.yml``:

.. code:: yaml

    name: Comment benchmark results

    on:
      workflow_run:
        workflows: ["Benchmark PR"]
        types: [completed]

    jobs:
      comment:
        if: >-
          github.event.workflow_run.event == 'pull_request' &&
          github.event.workflow_run.conclusion == 'success'
        runs-on: ubuntu-latest
        permissions:
          pull-requests: write
          issues: write
          actions: read

        steps:
          - name: Set up uv
            uses: astral-sh/setup-uv@v5

          - name: Download benchmark artifact
            uses: actions/download-artifact@v4
            with:
              name: benchmark-results
              path: results
              run-id: ${{ github.event.workflow_run.id }}
              github-token: ${{ secrets.GITHUB_TOKEN }}

          - name: Post benchmark comment
            uses: HaoZeke/asv-benchmark-commenter@v1
            with:
              github-token: ${{ secrets.GITHUB_TOKEN }}
              results-path: results/
              metadata-file: results/metadata.txt
              regression-threshold: '10'
              runner-info: ubuntu-latest

Required Permissions
--------------------

The commenter workflow needs:

.. table::

    +--------------------------+------------------------------------+
    | Permission               | Reason                             |
    +==========================+====================================+
    | ``pull-requests: write`` | Post/update PR comments            |
    +--------------------------+------------------------------------+
    | ``issues: write``        | Required for comment API           |
    +--------------------------+------------------------------------+
    | ``actions: read``        | Download artifacts from other runs |
    +--------------------------+------------------------------------+

What Happens Next
-----------------

When a pull request triggers the benchmark workflow and it succeeds:

1. The commenter workflow starts automatically

2. It downloads the benchmark results artifact

3. Reads SHAs from ``metadata.txt``

4. Finds the matching JSON files by SHA prefix

5. Runs ``asv-spyglass compare`` (or reads ``comparison.txt``)

6. Parses the output into regressions, improvements, unchanged

7. Renders a formatted comment with emoji indicators

8. Posts (or updates) the comment on the PR

9. Writes the same content to the workflow step summary

If the same PR is updated and benchmarks run again, the existing comment is
updated in place (idempotent via an HTML marker).
